# 🔍 Audit 13: Estratégia

**ROLE:** Analista de Negócios
**OBJETIVO:** Avaliar se o projeto tem direção clara e escalabilidade.

**INSTRUÇÃO:**
1. Busque `40_product_specs.md`, `ROADMAP.md`, ou issues/milestones documentados.
2. Avalie se o código atual reflete as specs (features implementadas vs planejadas).
3. Avalie onboarding: um novo dev entenderia o projeto em 15 minutos lendo a documentação?

**CRITÉRIO DE NOTA:**
- **10:** Roadmap claro, código alinhado, onboarding fácil
- **7-9:** Direção clara mas documentação de roadmap parcial
- **4-6:** Roadmap implícito, sem documentação formal
- **1-3:** Sem direção clara, sem documentação

## Regras

Siga as regras em `.agents/rules/common-patterns.md`.
